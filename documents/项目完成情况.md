## 四、各端完成情况明细

### 4.1 后端 API（`packages/server`）

**已完成：**

- NestJS 模块化：`auth`、`courses`、`surveys`、`stories`、`stats`、`team`、`admin`
- 全局：JWT 鉴权、`Public` / `OptionalAuth`、统一响应体、异常过滤、Swagger（`/api/docs`）
- Prisma 模型：用户、管理员、课程/课时、学习进度、问卷模板/提交、故事、邀请码、站点 KPI
- Seed：管理员、3 课程、2 问卷、2 故事、邀请码 `YUBING2026`、KPI 初始值

**未完成 / 待加强：**

| 项 | 说明 |
|----|------|
| 数据库迁移文件 | 仅有 `migration_lock.toml`，需本地执行 `pnpm db:migrate` 生成正式 migration |
| 微信登录 | API 已实现，需生产环境配置 `WECHAT_APPID` / `WECHAT_SECRET` |
| 管理端写操作 | 故事 CRUD、邀请码创建、问卷模板编辑、课时增删改 API 不完整 |
| 文件存储 | 无 COS 上传、预签名接口 |
| 导出格式 | 问卷仅 JSON，无 Excel/CSV 脱敏导出 |
| Redis | Docker 已配，业务未使用 |
| 自动化测试 | 仅保留 Nest 默认 e2e 占位，无业务测试 |

### 4.2 管理后台（`apps/admin`）

**已完成：**

- 登录 + 路由守卫（Pinia + JWT）
- 仪表盘（用户数、近 7 日问卷、KPI 展示）
- 课程：列表、新建、上下架
- 问卷：提交列表、JSON 导出
- 用户列表

**未完成：**

- 故事内容管理界面
- 邀请码管理
- 课程课时（章节）在线编辑

- 操作审计、权限细分（导师/超管菜单差异）

### 4.3 小程序（`apps/miniprogram`）

- 关怀模式入口与独立页

**未完成：**

|----|------|

## 七、风险与阻塞项

| 风险 | 影响 | 建议 |
|------|------|------|
| 未配置微信 AppSecret | 无法正式登录与上线审核 | 申请小程序并配置 `.env` |
| 无隐私政策与用户协议 | 审核可能驳回 | 撰写页面并后台可配置链接 |
| 数据库 migration 未固化 | 多人协作/部署易不一致 | 执行 `db:migrate` 并提交 migration 目录 |
| 内容依赖 Seed | 改内容需改代码或手动 DB | 完善后台 CMS + 批量导入 |
| 无自动化测试 | 回归成本高 | Phase 2 补充 API 集成测试 |
| Windows 下 `prisma generate` EPERM | 偶发构建失败 | 关闭占用进程或 `pnpm rebuild` |

## 八、建议后续优先级（路线图）

### 短期（1–2 周，补齐 MVP 可试点）

1. 配置微信登录与 `.env` 生产变量  
2. 执行并提交 Prisma migration；团队统一 `docker:up` + seed  
3. 隐私政策 / 用户协议（小程序页 + 后台链接）  
4. 后台：故事 CMS、课时编辑、邀请码管理  
5. 小程序：正式 `wx.login`、分享卡片、替换 Tab 图标  
6. 200+ 课时与媒体素材批量入库（脚本或飞书导入）

### 中期（Phase 2，3–6 周）

1. 学情前后测对比报表（教师/管理员）  
2. COS 视频上传 + 「仅 Wi-Fi 下载」  
3. 问卷 Excel 脱敏导出、订阅消息  
4. Redis 缓存与基础监控  

### 长期（Phase 3）

教具联动、文博 FAQ/大模型（合规后）、H5 成果专题页、挑战杯展示页。
