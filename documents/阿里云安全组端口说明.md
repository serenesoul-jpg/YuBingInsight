# 语冰 Insight — 阿里云安全组端口说明

> **当前公网 IP 示例**：`8.130.52.140`（变更后请同步更新本机 `YUBING_API_BASE`）  
> **配置位置**：阿里云控制台 → ECS 实例 → 安全组 → 入方向规则  
> **适用阶段**：MVP 开发联调（非生产 HTTPS 部署）

---

## 一、需要放行的端口（共 4 个）

开发阶段建议在安全组 **入方向** 放行以下端口。**来源** 优先填你的办公网 / 本机公网 IP，不要对全网开放 `0.0.0.0/0`（22 端口尤其注意）。

| 端口 | 协议 | 是否必需 | 作用 | 谁访问 | 对应命令 / 访问地址 |
|------|------|----------|------|--------|---------------------|
| **22** | TCP | **必需** | SSH 远程登录；Cursor Remote、终端运维 | 开发者 / 运维 | `ssh root@8.130.52.140` |
| **3000** | TCP | **必需** | NestJS API：小程序、H5、管理后台的数据接口 | 本机微信开发者工具、浏览器、H5 页面 | `pnpm dev:server`<br>`http://8.130.52.140:3000/api/v1`<br>Swagger：`/api/docs` |
| **10086** | TCP | **推荐** | Taro **H5 浏览器预览**（与小程序同源码，改完即看） | 开发者浏览器 | `pnpm dev:mp:h5`<br>`http://8.130.52.140:10086` |
| **5173** | TCP | **可选** | Vue **管理后台**开发服（课程、路径、视频链接 CMS） | 开发者浏览器 | `pnpm dev:admin`<br>`http://8.130.52.140:5173` |

### 按使用场景怎么选

| 你的目标 | 至少放行 |
|----------|----------|
| 只能 SSH 上服务器改代码 | 22 |
| 本机小程序连远程 API | 22 + **3000** |
| 浏览器看 H5 页面效果（服务器开发） | 22 + 3000 + **10086** |
| 浏览器改后台课程/内容 | 22 + 3000 + **5173**（10086 若同时做前端则一并开） |
| **完整开发联调（推荐）** | **22 + 3000 + 10086 + 5173** |

## 二、禁止对公网放行的端口

这些服务只应在本机 `127.0.0.1` 访问，**不要**在安全组开放。

| 端口 | 服务 | 原因 |
|------|------|------|
| **5432** | PostgreSQL（Docker `yubing-postgres`） | 数据库暴露公网风险极高；API 通过 `127.0.0.1:5432` 连接即可 |
| **6380** | Redis（Docker `yubing-redis`） | 当前 MVP API 未依赖 Redis；无业务必要不暴露 |
| **5555** 等 | Prisma Studio 等调试工具 | 仅本地临时使用，禁止长期公网暴露 |

## 四、阿里云控制台配置示例

**入方向规则**（每条规则示意）：

| 授权策略 | 端口 | 协议 | 来源 | 说明 |
|----------|------|------|------|------|
| 允许 | 22/22 | TCP | `你的IP/32` | SSH |
| 允许 | 3000/3000 | TCP | `你的IP/32` | API |
| 允许 | 10086/10086 | TCP | `你的IP/32` | H5 预览 |
| 允许 | 5173/5173 | TCP | `你的IP/32` | 管理后台 |

> 团队多人开发时，为每位开发者公网 IP 各加一条规则，或使用同一办公网 CIDR（如 `x.x.x.x/24`），仍避免 `0.0.0.0/0`。

## 五、放行后如何自检

在**你的本机**（非服务器上）执行：

```bash
# API
curl -s http://8.130.52.140:3000/api/docs -o /dev/null -w "API:%{http_code}\n"

# H5（需服务器已运行 pnpm dev:mp:h5）
curl -s http://8.130.52.140:10086 -o /dev/null -w "H5:%{http_code}\n"

# Admin（需服务器已运行 pnpm dev:admin）
curl -s http://8.130.52.140:5173 -o /dev/null -w "Admin:%{http_code}\n"
```

返回 `200` 或 `301/302` 一般表示端口可达；`000` 或超时多为安全组未放行或服务未启动。

在**服务器上**确认进程监听：

```bash
ss -tlnp | grep -E ':3000|:5173|:10086'
pnpm dev:server   # 若 3000 未监听
pnpm dev:mp:h5    # 若 10086 未监听
pnpm dev:admin    # 若 5173 未监听
```

## 六、与本机小程序联调的关系

| 组件 | 运行位置 | 需要开放的端口 |
|------|----------|----------------|
| 微信小程序编译 | **本机** `pnpm dev:mp` | 本机无需开端口；请求发往服务器 **3000** |
| API | **服务器** | **3000** |
| H5 预览 | **服务器** | **10086**（+ 3000 供页面调 API） |
| 管理后台 | **服务器** | **5173**（可选） |

本机 PowerShell 示例：

```powershell
$env:YUBING_API_BASE="http://8.130.52.140:3000"
pnpm dev:mp
```

## 七、后续生产环境（当前未启用）

正式对外上线时通常改为：

| 端口 | 用途 |
|------|------|
| **443** | HTTPS（Nginx 反向代理 API + 静态站） |
| **80** | HTTP 跳转 HTTPS（可选） |

届时 **3000 / 5173 / 10086** 应对公网关闭，仅内网或 Nginx 反代访问。本文档描述的是**当前 MVP 开发联调**规则。
