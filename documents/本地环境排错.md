# 本地环境排错指南

**Redis 端口冲突（6379 已被占用）：** 项目已改为映射 `6380`（容器内仍为 6379）。MVP 阶段 API **不依赖 Redis**，仅 Postgres 即可联调。执行 `pnpm docker:up` 重试，或忽略 Redis 失败直接 `pnpm db:push`。

**原因：** Docker Desktop 未安装或未启动。

**处理：**

1. 安装 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. 启动 Docker Desktop，等待右下角图标变为 **Running**
3. 再执行：

**没有 Docker 的替代方案：** 本机安装 PostgreSQL 15+，创建数据库 `yubing_insight`，修改 `.env` 中 `DATABASE_URL` 指向你的实例。

---

## 2. `DATABASE_URL` not found（Prisma migrate / seed）

**原因：** `.env` 在仓库根目录，但 Prisma 命令在 `packages/server` 下执行，默认读不到根目录 `.env`。

**处理：** 使用项目脚本（已配置 `dotenv-cli` 加载根目录 `.env`）：

```powershell
copy .env.example .env   # 若尚未创建
pnpm db:generate
pnpm db:push             # 或 pnpm db:migrate
pnpm db:seed
```

不要直接裸跑 `pnpm --filter @yubing/server exec prisma migrate dev`。

---

## 3. `bcrypt_lib.node` / Ignored build scripts: bcrypt

**原因：** pnpm 10 默认不执行部分原生模块编译；Windows 上 `bcrypt` 易出问题。

**处理：** 项目已改用纯 JS 的 `bcryptjs`，无需编译。若仍报错：

```powershell
pnpm install
pnpm db:seed
```

---

## 4. Prisma `EPERM: operation not permitted`（Windows）

**原因：** 其他进程占用 Prisma 引擎 DLL（如正在运行的 `dev:server`、IDE、杀毒软件）。

**处理：**

1. 关闭所有 `pnpm dev:server` 终端
2. 关闭微信开发者工具（若也在跑）
3. 重试：

```powershell
pnpm db:generate
```

仍失败时，删除缓存后重装：

```powershell
Remove-Item -Recurse -Force node_modules\.pnpm\@prisma+client@*\node_modules\.prisma -ErrorAction SilentlyContinue
pnpm install
pnpm db:generate
```

## 5. 推荐完整启动顺序（PowerShell）

```powershell
# 0. 启动 Docker Desktop 并等待就绪

cd D:\Users\liuch\Desktop\code\YuBingInsight
pnpm install
copy .env.example .env

pnpm docker:up
pnpm db:generate
pnpm db:push
pnpm db:seed

# 三个终端分别运行
pnpm dev:server
pnpm dev:admin
pnpm dev:mp
```

- 后台：http://localhost:5173 （admin / yubing2026）
- API：http://localhost:3000/api/docs
- 小程序：微信开发者工具打开**仓库根目录**，勾选「不校验合法域名」

## 6. 无数据库时能否预览 UI？

| 端 | 无数据库 |
|----|----------|
| 管理后台 | 可打开登录页，登录会失败 |
| 小程序 | 可看页面与 Logo，接口数据为空 |
| API | `/api/v1/health` 仍可用 |

完整联调仍需 PostgreSQL（Docker 或本机安装）。
