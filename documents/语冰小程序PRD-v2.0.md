## 一、文档目的与版本变更

### 1.1 目的

在 MVP（v1.0）已跑通「课程浏览 + 问卷 + 三大场景服务 + 关怀模式 + 后台管理」的基础上，**重新定义小程序的产品重心**：

| 维度 | v1.0（已交付 MVP） | v2.0（本版目标） |
|------|-------------------|------------------|
| 产品重心 | 基层场景工具 + 内容展示 | **人工智能课程体系**（冰块机器人 + 大学生 AI）+ **学习社区** |
| 首页角色 | 场景导航 + 故事墙 + KPI | 学习入口 + 社区动态 + **实践成果摘要** |
| 「服务」Tab | 乡村 / 社区 / 文博三大场景 | 降级为**课程专题 / 社区话题 / 成果专题**中的 contextual 入口 |
| 用户价值 | 「我能用什么工具」 | 「我能学什么、和谁交流、项目做成了什么」 |
| 数据目标 | 问卷与调研为主 | **学习行为 + 社区互动 + 调研**三线并行 |

### 1.2 不变项（继承 v1.0）

- 品牌：语冰 AI 普惠，Slogan「让 AI 既有登高的硬度，也有温暖民心的温度」
- 技术栈：Taro 3 + NestJS + PostgreSQL + Vue Admin（见 [技术栈确认.md](./技术栈确认.md)）
- 关怀模式：面向老人/残障用户的无障碍体验**保留并嵌入学习场景**
- 实践调研：队员邀请码、问卷、离线同步**保留**，作为社区/课程的数据闭环能力
- 管理后台：内容 CMS、问卷导出、用户管理**延续扩展**

## 二、产品定位

### 2.1 一句话定义

**语冰**是一款以**人工智能学习**为核心的微信小程序：围绕**冰块机器人**与**大学生 AI** 两大课程体系提供图文/视频内容与学习路径，以**学习社区**促进交流共创，并以**实践成果馆**展示语冰实践队的真实落地成效。

### 2.2 产品类型

公益导向 · AI 学习平台 · 轻量社区 · 实践成果展示

### 2.3 核心价值主张

| 对象 | 价值 |
|------|------|
| 大学生 / 青年学习者 | 按 AI 学习路径系统学习，观看视频/图文，在社区提问 |
| 教师 / 基层实践者 | 学习冰块机器人教案，跟练硬件教学场景，获得可验证进度 |
| 社区参与者 | 分享学习笔记、提问答疑、围观实践故事，降低孤独感 |
| 实践队员 | 在统一平台发布一线素材、发起话题、回收调研 |
| 项目方 / 媒体 | 成果数据与故事可检索、可分享，支撑路演与报告 |
| 管理员 | 课程与社区内容可运营，学习数据与调研数据可导出 |

### 2.4 设计原则

1. **学习优先**：首屏 3 秒内让用户看到「继续学习」或「推荐路径」
2. **社区有温度**：UGC 以学习相关为主，避免泛社交；强调互助而非流量
3. **成果可信**：实践成果区数据可溯源，故事与 KPI 来自后台 CMS，非硬编码
4. **低门槛 + 无障碍**：关怀模式、大字号、语音辅助在学习与社区场景均可用
5. **轻量可维护**：首包 < 2MB；非核心页分包；弱网离线缓存
6. **Web 优先开发**：同一套 Taro 源码先在浏览器（H5）迭代 UI，再打包为微信小程序

## 三、已确认产品决策

以下事项已在 2026-06-28 产品讨论中**确认**，后续设计与开发以此为准：

| 决策项 | 结论 | 说明 |
|--------|------|------|
| 硬件课程对外名称 | **统一叫「冰块机器人」** | 不再对外混用「语冰机器人课」；代码常量可保留 `ice-robot`，UI 文案统一 |
| 大学生 AI 首期路径规模 | **节点数 > 4（建议 6～8 个）** | 含课程课时、图文阅读、阶段测验/问卷等；保证路径有「系统感」而非 demo |
| 视频托管（首期） | **HTTPS 外链** | 后台课时填 `videoUrl`；团队可先用 B 站/腾讯系 CDN/学校存储等稳定 HTTPS 地址 |
| 视频托管（上线前） | **逐步迁 COS** | 微信小程序正式发布前，将高频视频迁至腾讯云 COS + CDN，并配置合法域名 |
| 开发主战场 | **阿里云服务器** | API + 管理后台 + **H5 浏览器预览**在服务器；本机仅 `git pull` + 微信开发者工具验收 |
| 源码同步 | **GitHub** | 服务器改代码 → push → 本机 pull → `pnpm dev:mp` 做小程序端最终确认 |

### 3.1 视频外链 vs COS：推荐路径

| 阶段 | 方案 | 理由 |
|------|------|------|
| **现在～内容筹备** | HTTPS 外链 | 上传快、无需等 COS 开通；H5 预览即改即看；适合团队并行产内容 |
| **联调后期** | 外链 + 抽测 COS | 验证 COS 上传链路、域名配置流程 |
| **提交微信审核前** | 核心课迁 COS | 小程序 `video` 组件对域名有要求；外链平台可能限流或防盗链导致播放失败 |
| **长期运营** | COS 为主、外链为辅 | 大文件、更新频的内容走 COS；外链仅作补充或跳转 |

**外链使用规范**：

- 必须 **HTTPS**，避免 mixed content
- 优先直链 MP4 或微信/腾讯生态可稳定播放的地址；避免需 Cookie 的网盘分享页
- 后台字段：`videoUrl`（外链）+ 可选 `videoProvider`（`external` / `cos`）便于后续迁移
- H5 开发阶段可直接播放；微信端上线前在「开发设置 → 服务器域名」配置 downloadFile 合法域名

### 3.2 首期「大学生 AI 通识入门」路径（建议 8 节点）

> 名称与具体内容待团队上传素材后微调；**节点总数不少于 5，建议 8 个**。

| 序号 | 节点类型 | 节点标题（示例） |
|------|----------|------------------|
| 1 | 视频课时 | 什么是人工智能 |
| 2 | 图文阅读 | AI 在你身边的 10 个例子 |
| 3 | 视频课时 | 大模型能做什么、不能做什么 |
| 4 | 实操图文 | Prompt 入门：写好第一条指令 |
| 5 | 视频课时 | 常用 AI 工具上手（演示） |
| 6 | 小测验 | 章节自测（5 题） |
| 7 | 视频/图文 | AI 伦理与负责任使用 |
| 8 | 问卷 | 路径前/后测（复用 Survey） |

## 五、开发与预览策略（Web 优先 · 服务器开发）

### 5.1 能否「网页先看效果，再封装小程序」？

**可以。** 本项目使用 **Taro 3**，同一套 React 源码可编译为：

| 目标 | 命令（仓库根目录） | 输出 / 访问 |
|------|-------------------|-------------|
| **H5 网页（开发推荐）** | `pnpm dev:mp:h5` | 浏览器访问 `http://<服务器IP>:10086` |
| 微信小程序 | `pnpm dev:mp` | 本机微信开发者工具打开仓库根目录 |
| 管理后台 | `pnpm dev:admin` | `http://<服务器IP>:5173` |
| API | `pnpm dev:server` | `http://<服务器IP>:3000/api/v1` |

**推荐分工**：

```text
┌─────────────────────────────────────────────────────────────────┐
│  阿里云服务器（主开发）                                           │
│  pnpm dev:server   → :3000   API                                 │
│  pnpm dev:admin    → :5173   后台 CMS（改课程/路径/视频链接）      │
│  pnpm dev:mp:h5    → :10086  浏览器预览小程序 UI（改完即看）        │
└───────────────────────────────┬─────────────────────────────────┘
                                │ git push / pull
┌───────────────────────────────▼─────────────────────────────────┐
│  开发者 Windows 本机（验收）                                      │
│  git pull → pnpm dev:mp → 微信开发者工具预览 weapp                  │
│  （仅在做分享、登录、TabBar 等微信特有能力验收时必需）               │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 安全组需放行端口

| 端口 | 用途 | 建议来源 |
|------|------|----------|
| 3000 | API | 开发 IP |
| 5173 | 管理后台 | 开发 IP |
| **10086** | **H5 小程序预览** | 开发 IP |
| 22 | SSH | 办公网 |

**不要**对公网开放 5432。H5 预览建议限制来源 IP，避免未授权访问开发态页面。

### 5.3 H5 与微信小程序差异（验收时注意）

| 能力 | H5 预览 | 微信小程序 |
|------|---------|------------|
| 页面布局 / 路由 / 大部分 UI | ✅ 高度一致 | ✅ |
| 调 API（同服务器） | ✅ | ✅（需配置 request 合法域名或开发期不校验） |
| `wx.login` 正式登录 | ⚠️ 不可用或模拟 | ✅ |
| 分享朋友圈 / 卡片 | ⚠️ 行为不同 | ✅ 需本机验收 |
| TabBar / 原生组件细节 | ⚠️ 略有差异 | ✅ 以微信为准 |
| 视频外链 | ✅ 浏览器友好 | ⚠️ 需配置域名 + 正式环境测播放 |

**结论**：日常 UI、课程流、社区流、成果馆在 **H5 上开发效率最高**；**微信开发者工具仅做发布前验收**（建议每 Sprint 至少 1 次）。

### 5.4 环境变量（H5 联调远程 API）

服务器上 H5 与 API 同机时，默认 `API_BASE=http://localhost:3000` 即可。  
若 H5 在本机浏览器访问远程服务器页面，需构建时指定：

```bash
YUBING_API_BASE=http://8.130.52.140:3000 pnpm dev:mp:h5
```

### 5.5 首次启用 H5 开发（工程 checklist）

- [ ] 安装依赖：`pnpm install`（若缺 H5 插件：`pnpm --filter @yubing/miniprogram add -D @tarojs/plugin-platform-h5@3.6.34`）
- [ ] 根目录执行：`pnpm dev:mp:h5`
- [ ] 浏览器打开：`http://8.130.52.140:10086`
- [ ] 并行：`pnpm dev:server`、`pnpm dev:admin`
- [ ] 改 `apps/miniprogram/src` 保存后 H5 热更新
- [ ] 阶段性 `git push` → 本机 `git pull` → `pnpm dev:mp` 微信验收

**相对 v1.0 变更**：

| v1.0 Tab | v2.0 处理 |

## 八、核心模块需求

### 8.1 人工智能课程体系

#### 8.1.1 目标

将学习 Tab 打造为**以人工智能为主题**的内容与学习中枢，而非泛化的「数字素养」或场景工具集合。用户进入后首先选择两大模块之一，在模块内浏览/学习由团队上传的**图文与视频**，大学生模块 additionally 通过**学习路径**获得结构化进阶体验。

#### 8.1.2 两大核心模块

学习 Tab 首屏展示两个并列模块卡片（可横向滑动或上下排列）：

```
┌─────────────────────────────────────────────────────────┐
│  继续学习：大学生 AI 路径 · 第 2 课 / 共 8 课            │
├──────────────────────────┬──────────────────────────────┤
│  🤖 冰块机器人课程体系    │  🎓 大学生 AI 课程体系        │
│  自研教具 · 实操教学      │  学习路径 · 视频/图文         │
│  图文 + 视频              │  面向高校 AI 通识与进阶       │
└──────────────────────────┴──────────────────────────────┘
```

##### 模块一：冰块机器人课程体系

| 项 | 说明 |
|----|------|
| **定位** | 语冰自研**冰块机器人**教具的配套 AI 教学课程：从认识硬件、基础操作到课堂应用 |
| **目标用户** | 基层教师、实践队员、带队老师、对硬件 AI 教学感兴趣的用户 |
| **内容形态** | **图文**（操作步骤、教案、示意图）+ **视频**（演示、课堂实录）；由管理员在后台上传 |
| **组织方式** | 按**主题/难度**分课程 → 每课含多个**课时（Unit）**；首期可不强制路径，以课程列表为主 |
| **AI 相关度** | 强调「AI 硬件教具如何进课堂」：感知、交互、编程启蒙、人机协作等，而非纯工具说明 |
| **与硬件关系** | 不替代实体机器人；课程内可含「扫码绑定教具」（Phase 2）、课堂脚本、故障排查 |

**首期内容规划（待团队上传）**：

- 冰块机器人认识与开箱
- 基础动作与交互演示
- 课堂组织与教学脚本
- 常见问题与维护（图文为主）

##### 模块二：大学生 AI 课程体系

| 项 | 说明 |
|----|------|
| **定位** | 面向高校学生的 **AI 通识与进阶** 线上课程：概念、应用、工具、伦理与实践 |
| **目标用户** | 大学生（含武大实践队员）、青年学习者、对 AI 有系统学习需求的公众 |
| **内容形态** | **视频**（讲座、实操录屏）+ **图文**（课件、阅读材料、习题解析）；由管理员在后台上传 |
| **组织方式** | **学习路径（Roadmap）为核心** + 课程列表辅助浏览；路径内节点有序，展示完成度 |
| **AI 相关度** | 全模块围绕人工智能：机器学习直觉、大模型应用、Prompt、AI 伦理、行业案例等 |
| **学习路径** | 至少 1 条官方路径上线（可扩展多条专题路径） |

**建议首期学习路径示例**（内容待上传，名称可调整）：

| 路径 ID | 名称 | 节点示例 | 说明 |
|---------|------|----------|------|
| `path-college-ai-intro` | AI 通识入门 | **8 节点**（见 §3.2） | **P0 首发路径，节点 >4** |
| `path-college-ai-practice` | AI 应用实践 | 工具选型 → 案例分析 → 小组作业指引 → 成果展示 | P1 |
| `path-college-ai-ethics` | AI 伦理与社会 | 偏见与公平 → 隐私 → 就业影响 → 讨论题 | P1 |

**单课 / 单节规范**：

- 视频课：建议 5～15 分钟/节；**首期 `videoUrl` 填 HTTPS 外链**（见 §3.1）；上线前核心课迁 COS
- 图文课：步骤清晰，配图可缓存；关怀模式下字号放大
- 每课至少包含：标题、简介、封面、课时列表、所属模块标签

#### 8.1.3 内容上传与 CMS（团队侧）

团队将通过**管理后台**维护两大模块内容（无需改代码发版）：

| 能力 | P0 | 说明 |
|------|----|------|

| 视频课时 | ✅ | **HTTPS 外链 `videoUrl`（首期）**；P1 起支持 COS 上传 |
| 学习路径编排 | ✅ | 大学生模块：路径节点拖拽排序，关联课程/问卷 |
| 上下架 | ✅ | 课程/课时/路径独立发布状态 |
| 批量导入 | P1 | 飞书表格 / Excel 批量导入课时元数据 |

## 十三、API 扩展概要

在现有 `/api/v1` 基础上新增（命名示例）：

```
GET  /learning-paths?module=college-ai

GET  /learning-paths/:id/progress
GET  /courses?module=ice-robot|college-ai
GET  /courses/:id/units/:unitId          # 课时详情（图文 body / videoUrl）
PATCH /courses/units/:unitId/progress    # 沿用
POST /community/posts
GET  /community/posts
GET  /community/posts/:id
POST /community/posts/:id/comments
POST /community/posts/:id/like
GET  /community/topics
GET  /impact/summary          # 聚合 stats + 精选故事
```

现有 `courses`、`surveys`、`stories`、`stats`、`team` **保持兼容**；v2.0 以前端 IA 重组为主，后端增量开发。

## 十四、分期实施计划

### Phase 2A — 双模块课程体系（约 3～4 周）

**目标**：学习 Tab 以两大 AI 模块上线；团队可通过后台上传图文/视频；大学生模块具备学习路径。

- 数据模型：`Course.module`、`CourseUnit.contentType` / `videoUrl` / `body`
- 学习 Tab 双模块入口 + 模块首页
- 冰块机器人：课程列表 / 详情 / 课时（图文+视频）
- 大学生 AI：路径列表 / 路径详情 / 课程课时 + 进度
- 后台：模块选择、课时编辑、路径节点编排
- 首页「继续学习」对接最近课时或路径节点

**验收**：

- 管理员上传 1 门冰块机器人课（≥ 2 课时）+ 1 条大学生 AI 路径（**> 4 节点，建议 8**）后可完整演示
- **H5 预览**：`http://<IP>:10086` 可浏览学习 Tab 双模块
- 用户完课进度可回显

### Phase 2B — 学习社区（约 3 周）

**目标**：社区 Tab 上线；课程与社区打通。

- Post / Comment / Topic 模型与 API
- 社区 Feed、发帖、详情、点赞
- 内容安全（msgSecCheck）
- 后台帖子审核

**验收**：每日可产生可审核 UGC；课时页可跳转讨论。

### Phase 2C — 实践成果馆（约 2 周）

**目标**：v1.0 传播能力保留并集中展示。

- 成果馆二级页
- 三场景页迁移
- 分享卡片 / 海报
- 首页 KPI 区块优化

**验收**：路演时可仅演示「学习 + 社区 + 成果馆」30 分钟闭环。

### Phase 3 — 智能化与运营（后续）

- 学情前后测报表（教师/管理员）
- 笔记、关注、AI FAQ
- 订阅消息（课程更新、回复提醒）
- COS 视频、Wi-Fi 下载
- 五省六地地图、赛事专题页

## 十六、风险与对策

| 风险 | 对策 |
|------|------|
| 视频外链失效 / 防盗链 | 首期外链 + 上线前核心课迁 COS；后台支持一键改 URL |
| H5 与微信表现不一致 | 每 Sprint 本机微信验收 1 次；差异项记入已知问题列表 |
| 社区 UGC 合规 | 先发后审 + 敏感词 + 举报；官方内容为主冷启动 |
| 学习与服务 Tab 取消后老用户困惑 | 成果馆内保留场景入口；社区话题承接讨论 |
| 内容冷启动 | 队员与管理员种子帖；优质故事同步到社区 |
| 开发资源有限 | 严格 P0 边界；路径 3 条即可上线 |
| AI 能力合规 | Phase 3 再做；一期仅 FAQ 检索 |
