# 语冰 AI 普惠 — 项目理解与技术栈说明

> 本文档基于武汉大学计算机学院 E乡情·语冰「人工智能+」基层赋能实践队项目《语冰·AI普惠的「最后一公里」》PRD（v1.0，2026-06-27），结合当前仓库现状整理，供产品、设计、开发协同使用。

---

## 一、项目要做什么（一句话）

**语冰 AI 普惠**是武汉大学 E乡情·语冰实践队的公益小程序：把自研教具（编程拼图、语冰机器人）、课程与调研能力，通过轻量小程序触达**乡村教师、社区老人、文博场景、实践队员**等基层场景，同时支撑路演演示、试点教学与调研数据沉淀。

它不是硬件替代品，而是**连接、记录、迭代、传播**的数字探针。

---

## 二、产品定位与设计原则

| 维度 | 说明 |
|------|------|
| 产品名称 | 语冰 AI 普惠（简称「语冰」） |
| Slogan | 让 AI 既有登高的硬度，也有温暖民心的温度 |
| 产品类型 | 公益导向 + 轻量工具 + 内容服务的 B2G2C 小程序 |
| 核心价值 | 将高校自研教具、课程与调研能力，通过小程序触达基层三类场景 |
| 与硬件关系 | 不替代编程拼图与语冰机器人；负责连接、记录、迭代、传播 |

### 设计原则

- **首屏 3 秒可懂**：新用户打开即见「我能做什么」+ 一键进入场景
- **低门槛**：大字号、高对比、语音辅助；支持「关怀模式」一键切换
- **轻量启动**：首包 < 2MB，非核心页分包加载
- **离线友好**：乡村弱网下课程与问卷支持本地缓存、断点续传
- **隐私合规**：最小必要采集，敏感数据加密，符合《个人信息保护法》及微信隐私协议

## 四、产品形态与信息架构

采用 **4 Tab + 1 全局悬浮入口** 结构：

```
┌─────────────────────────────────────────────────┐
│  [首页]  [学习]  [服务]  [我的]                  │
│           ↑ 全局悬浮：语音助手 / 人工客服        │
└─────────────────────────────────────────────────┘
```

### Tab 1 — 首页（场景导航）

- 品牌区：语冰理念、项目进展（五省六地地图热力）
- 三大场景入口：乡村 AI 课堂 / 社区数字助老 / 文博 AI 活化
- 快捷工具：扫码绑定教具、今日课程、附近活动
- 故事墙：媒体报道、实践纪实短视频

### Tab 2 — 学习（内容与课程）

| 模块 | 功能说明 |
|------|----------|
| 编程拼图学堂 | 分层次课程（图文 + 短视频 + 无屏操作指引） |
| 语冰机器人课 | 情景教案、互动问答示例、课堂脚本 |
| 珞恩成长课堂 | 数智科普讲座回放、课件下载 |
| AI 素养通识 | 面向老人/家长的轻量科普内容 |
| 学习进度 | 课时完成、前后测入口（对接调研数据） |

规范：课程单元 ≤ 5 分钟/节；支持「仅 Wi-Fi 下载」；进度云同步。

### Tab 3 — 服务（基层赋能工具）

- **乡村 AI 教育**：教具手册、教师备课包下载、学情问卷/儿童前后测、申请支教表单
- **社区助老助残**：关怀模式入口、场景化设备指南、一键呼叫志愿者、无障碍设置
- **文博活化**：场馆 AI 导览、文物故事互动、研学任务卡与打卡勋章
- **调研探针（队员端）**：问卷分发回收（支持离线）、访谈录音上传、GPS 签到（可选）

### Tab 4 — 我的

- 微信一键登录 + 按需手机号绑定
- 角色切换：家长 / 教师 / 老人 / 队员 / 管理员
- 学习记录、勋章、收藏
- 隐私中心、意见反馈、关于语冰

## 五、关怀模式（差异化体验）

面向老人与残障用户的独立 UI 主题，非仅放大字体：

| 设计项 | 规范 |
|--------|------|
| 字号 | 默认 20px+，最大 28px 可调 |
| 色彩 | 高对比（WCAG AA），避免纯蓝链接 |
| 交互 | 单屏单任务，底部固定「返回首页」「语音朗读」 |
| 输入 | 优先语音输入（微信插件或自建 ASR） |
| 导航 | 最多 3 个主按钮，文字 + 图标双编码 |
| 错误提示 | 用人话说明，提供「找志愿者帮忙」入口 |

入口：首页 Banner「长辈模式」+ 设置内永久开关 + 子女代开（分享卡片带参数）。

## 六、核心用户流程

### 6.1 乡村教师首次使用

扫码打开 → 选择「我是教师」→ 绑定学校/地区 → 下载本周课程包 → 查看编程拼图教案 → 课后提交学情反馈 → 获得「语冰园丁」勋章。

### 6.2 老人学习「用手机挂号」

子女分享关怀模式链接 → 大按钮「学挂号」→ 分步图文（每步可朗读）→ 模拟练习 → 完成打卡 → 可选联系社区志愿者。

### 6.3 实践队员田野调研

队员认证（邀请码）→ 选择调研模板（乡村/社区/文博）→ 现场问卷/访谈 → 上传素材 → 自动汇总至后台看板 → 支撑报告产出。

### 6.4 公众了解项目（传播）

首页故事墙 → 纪实视频 → 分享朋友圈/群 → 落地页展示成果数据（200+ 课时、14 家媒体报道等）。

## 七、功能分期与实现范围

### Phase 1 — MVP（4–6 周）

目标：可演示、可收集数据、支撑路演与试点。

**建议双主线：乡村课堂 + 关怀模式。**

| 优先级 | 功能项 | 验收标准 |
|--------|--------|----------|
| P0 | 三大场景首页 + 关怀模式 | 首屏加载 < 1.5s；关怀模式一键切换 |
| P0 | 课程列表与详情（静态 + 视频） | 至少 3 套课程包可浏览；支持 Wi-Fi 下载 |
| P0 | 微信登录 + 角色选择 + 个人中心 | 5 类角色菜单差异化展示 |
| P0 | 简化问卷（乡村学情 / 老人数字能力前测） | 支持离线填写、联网同步；匿名选项 |
| P1 | 队员调研录入 | 邀请码认证；三类调研模板 |
| P1 | 管理后台 Web | 问卷汇总、用户与内容管理（可与小程序分离） |
| P1 | 故事墙 + 成果数据 + 分享卡片 | 可分享至微信；展示关键 KPI |

### Phase 2 — 数据闭环

- 学情前后测对比报表（教师/管理员可见）
- 课程与教具反馈 → 驱动迭代建议
- 调研数据库导出（脱敏），支撑 1.5 万字主报告
- 订阅消息：活动提醒、课程更新

### Phase 3 — 智能化与生态

- 语冰机器人 / 编程拼图设备扫码或蓝牙联动（视硬件能力）
- 文博 AI 问答（需内容审核与备案）
- 外部 AI 社区资源合规外链
- 挑战杯 / 互联网+ 成果展示专题页

## 九、技术栈（已确认 v1.1）

> 详细论证、目录规划与分期落地见 **[技术栈确认.md](./技术栈确认.md)**。

### 9.1 确定方案一览

| 层级 | 确定方案 |
|------|----------|
| **仓库** | pnpm Monorepo（`apps/miniprogram`、`apps/admin`、`packages/server`、`packages/shared`） |
| **小程序** | Taro 3 + React 18 + TypeScript + NutUI React |
| **开发者后台** | Vue 3 + Vite + Element Plus |
| **后端 API** | NestJS 10 + TypeScript + Prisma |
| **数据库** | PostgreSQL 15+ |
| **缓存** | Redis（Phase 2） |
| **对象存储** | 腾讯云 COS + CDN |
| **接口** | REST + OpenAPI 3；JWT 鉴权 + RBAC |
| **内容管理** | 后台内置 CMS（飞书表格仅作批量导入工具） |
| **部署** | Docker + Nginx |
| **微信云开发** | **不作为长期主后端**（当前 QuickStart 可弃用，避免双写） |

### 9.2 架构示意

```
微信小程序 (Taro)  ──┐
                     ├──▶  NestJS API  ──▶  PostgreSQL
开发者后台 (Vue3)  ──┘              └──▶  COS + CDN
```

### 9.3 选型要点（长期维护 + 后台）

- **趁无业务代码迁移 Taro**：成本接近从零搭建，换来 TS、组件化与可选 H5。
- **NestJS + PostgreSQL**：问卷汇总、脱敏导出、RBAC、报表是后台核心，关系型库 + 正规 API 比云开发更合适。
- **Vue Admin 与 Taro 解耦**：管理端用 Element Plus 生态成熟，不与小程序强行同框架。
- **统一 API**：小程序与后台只调 NestJS，禁止云开发与自建 API 双写同一业务表。

### 9.4 当前仓库状态

| 项 | 现状 |
|----|------|
| 框架 | Taro 3 + React + TS（`apps/miniprogram`） |
| 后台 | Vue 3 + Element Plus（`apps/admin`） |
| API | NestJS + Prisma（`packages/server`） |
| 完成度 | 见 **[项目完成情况.md](./项目完成情况.md)** |

## 十、安全与合规要点

- **儿童信息**：默认不采集生物识别；问卷提供匿名；需家长知情同意
- **调研录音**：明确告知用途与保存期限
- **管理端 RBAC**：队员 / 导师 / 超管分级
- **UGC**：走微信 `msgSecCheck`；视频先审后发

## 十四、风险与对策

| 风险 | 对策 |
|------|------|
| 乡村网络差 | 离线包、图文优先、压缩视频 |
| 老人不会用小程序 | 关怀模式 + 子女代操作 + 线下队员引导 |
| 功能过多臃肿 | 严格 MVP，按角色隐藏菜单 |
| AI 对话合规 | 一期预设 FAQ；大模型二期需类目与审核 |
| 维护人力不足 | 课程 CMS 化，减少发版频率 |
