## 一、决策结论（可直接执行）

| 层级 | 确定方案 | 说明 |
|------|----------|------|
| **仓库组织** | **pnpm Monorepo** | 小程序、后台、共享类型同一仓库，版本与接口对齐 |
| **小程序** | **Taro 3 + React 18 + TypeScript** | 组件化、类型安全、可扩 H5；趁无业务代码时迁移成本最低 |
| **小程序 UI** | **NutUI React（Taro 版）** + 自定义关怀模式主题 | 贴近微信视觉，适老化组件可自建 |
| **开发者后台** | **Vue 3 + Vite + Element Plus** | 管理端生态成熟，表格/表单/权限脚手架多，与小程序解耦是行业常态 |
| **后端 API** | **NestJS 10 + TypeScript** | 模块化、RBAC、OpenAPI、适合多人接力维护 |
| **数据库** | **PostgreSQL 15+** | 问卷/学情/用户关系型建模，报表导出、脱敏比文档库更合适 |
| **缓存** | **Redis**（Phase 2 引入即可） | 会话、热点课程、统计聚合 |
| **对象存储** | **腾讯云 COS + CDN** | 课件、视频、访谈录音；与微信生态一致 |
| **接口规范** | **REST + OpenAPI 3** | 小程序与后台共用同一套 API；`packages/shared` 共享类型 |
| **鉴权** | 小程序 `wx.login` → 后端 `code2Session` → **JWT**；后台账号 + JWT | RBAC：队员 / 导师 / 超管 |
| **内容管理** | **后台内置 CMS**（课程、故事、问卷模板、媒体） | 飞书表格仅作 Phase 1 批量导入工具，不作长期依赖 |
| **部署** | **Docker + Nginx** | 武大校内服务器或腾讯云；环境用 `.env` 区分 |
| **微信云开发** | **不作为长期主后端** | QuickStart 代码可弃用或仅保留云存储过渡；数据与逻辑统一走 NestJS |

## 二、为什么调整（相对 v1.0「原生 + 云开发」）

| 诉求 | 原生 + 云开发的问题 | 新方案如何满足 |
|------|---------------------|----------------|
| **长期维护** | 原生 WXML/JS 难复用、无类型、多人协作易冲突 | Taro + TS + 组件库；NestJS 分层清晰 |
| **开发者后台** | 云数据库无友好管理界面；云函数不适合复杂 RBAC/报表 | Element Plus 后台 + PostgreSQL 查询/导出 |
| **调研数据自主可控** | 云开发数据在校外微信环境，导出与权限审计受限 | 自建库 + 脱敏导出 API，符合 PRD 8.2 |
| **内容迭代** | 改课程需发版或依赖飞书同步脚本 | 后台 CMS 改内容，小程序拉接口，减少发版 |
| **队员更替** | 原生 + 云函数知识分散 | 主流 Web 技术栈，文档与招聘友好 |

**关键时机：** 当前仅有 QuickStart 演示页，**现在选型 Taro 的迁移成本 ≈ 从零搭 Taro**，远小于 Phase 2 从原生重写。

## 三、总体架构

```
                         ┌─────────────────────────────────────┐
                         │           腾讯云 COS + CDN           │
                         │     视频 / 课件 PDF / 访谈录音        │
                         └──────────────────┬──────────────────┘
                                            │
    ┌──────────────────┐                    │                    ┌──────────────────┐
    │  微信小程序       │   HTTPS / REST     │                    │  开发者后台 Web   │
    │  Taro3 + React   │◀───────────────────┼───────────────────▶│  Vue3 + Element  │
    │  + TypeScript    │                    │                    │  Plus            │
    │  · 4 Tab + 分包   │                    ▼                    │  · CMS / 看板    │
    │  · 关怀模式       │         ┌─────────────────────┐         │  · 问卷汇总导出   │
    │  · 离线草稿同步   │         │   NestJS API        │         │  · RBAC 权限     │
    └──────────────────┘         │   · 微信登录          │         └──────────────────┘
            │                    │   · 课程/问卷/用户    │
            │ wx.login           │   · 文件上传签名      │
            └───────────────────▶│   · 统计 / 导出       │
                                 └──────────┬──────────┘
                                            │
                    ┌───────────────────────┼───────────────────────┐
                    ▼                       ▼                       ▼
              PostgreSQL              Redis (P2)              微信开放平台
              用户/角色/课程            缓存/限流                 msgSecCheck 等
              问卷/学情/调研
```

**数据流原则：**

1. 小程序与后台 **只通过 NestJS API 读写业务数据**，不直连数据库。
2. 大文件走 **COS 直传**（后端签发临时密钥），API 只存 URL 与元数据。
3. 离线问卷：小程序 `localStorage` 草稿 → 联网后 **批量同步接口**（带 `draftId` 幂等）。

## 四、Monorepo 目录规划（目标结构）

```
YuBingInsight/
├── apps/
│   ├── miniprogram/          # Taro 3 小程序（微信开发者工具打开 dist 或配置 project.config）
│   └── admin/                # Vue3 管理后台
├── packages/
│   ├── shared/               # 共享：枚举、DTO 类型、校验规则（zod）
│   └── server/               # NestJS 后端（或命名为 api）
├── documents/                # 产品与技术文档
├── docker/
│   ├── docker-compose.yml    # postgres + redis + api + admin(nginx)
│   └── nginx.conf
├── pnpm-workspace.yaml
└── package.json
```

**微信开发者工具：** `project.config.json` 的 `miniprogramRoot` 指向 `apps/miniprogram/dist`（Taro 构建产物）。

**过渡期：** 现有根目录 `miniprogram/`、`cloudfunctions/` 在完成 Taro 脚手架后可删除或移至 `legacy/`。

## 五、各端技术明细

### 5.1 小程序（apps/miniprogram）

| 项 | 选型 |
|----|------|
| 框架 | Taro 3.6+ |
| 语言 | TypeScript 5.x |
| UI | NutUI React @tarojs/taro |
| 状态 | Zustand（轻量）或 React Context + hooks |
| 请求 | Taro.request + 封装（JWT、错误码、重试） |
| 分包 | 主包：首页/学习/服务/我的；分包 `team`：队员调研 |
| 关怀模式 | 全局 `careMode` store + CSS 变量主题 + `/pages/care/*` 独立页 |
| 离线 | `Taro.setStorage` 草稿队列 + 同步服务 |
| 统计 | 微信小程序数据助手 + 自定义事件上报 API |

### 5.2 开发者后台（apps/admin）

| 项 | 选型 |
|----|------|
| 框架 | Vue 3 + Vite 5 |
| UI | Element Plus |
| 路由 | Vue Router 4 |
| 状态 | Pinia |
| 请求 | Axios + OpenAPI 生成客户端（可选） |
| 权限 | 动态路由 + 按钮级 `v-permission` |

**后台功能模块（与 PRD P1 对齐）：**

| 模块 | 功能 |
|------|------|
| 仪表盘 | DAU、问卷数、完课率、关怀模式占比、五省六地分布 |
| 用户与角色 | 用户列表、角色分配、邀请码管理 |
| 课程 CMS | 课程包 / 章节 / 视频链接 / 上下架 |
| 问卷管理 | 模板编辑、回收数据、导出 CSV/Excel（脱敏） |
| 内容运营 | 故事墙、成果 KPI、Banner |
| 调研素材 | 访谈录音列表、审核状态 |
| 系统 | 管理员账号、操作日志、隐私相关配置 |

### 5.3 后端 API（packages/server）

| 项 | 选型 |
|----|------|
| 框架 | NestJS 10 |
| ORM | Prisma（或 TypeORM，推荐 Prisma 迁移清晰） |
| 校验 | class-validator + class-transformer |
| 文档 | @nestjs/swagger → OpenAPI |
| 鉴权 | @nestjs/passport + JWT；微信 `jscode2session` |
| 文件 | COS SDK + 预签名上传 |
| 任务 | Bull + Redis（异步导出、定时统计，P2） |
| 日志 | Winston + 文件/stdout |

**核心数据域（PostgreSQL）：**

```
users          # 微信 openid、手机号(可选)、角色、关怀模式偏好
roles          # parent / teacher / elder / member / admin
courses        # 课程包
course_units   # 章节/课时
learning_progress
survey_templates / survey_questions
survey_submissions  # 支持匿名标记
stories        # 故事墙
devices        # 教具扫码绑定（P3）
invite_codes   # 队员邀请码
audit_logs     # 管理操作审计
```

### 5.4 基础设施

| 项 | 选型 |
|----|------|
| 运行环境 | Node.js 20 LTS |
| 包管理 | pnpm 9 |
| 容器 | Docker Compose（开发/部署一致） |
| 反向代理 | Nginx（HTTPS、静态后台、API 转发） |
| CI | GitHub Actions / Gitee Go（lint、test、build） |
| 密钥 | 环境变量 + 不入库；生产用服务器密钥管理 |

## 六、微信云开发定位（明确边界）

| 能力 | 长期方案 | 云开发 |
|------|----------|--------|
| 业务数据库 | PostgreSQL | ❌ 不用于核心业务 |
| 文件存储 | COS | ⚠️ 可与云存储并存，统一迁 COS |
| 云函数 | NestJS API | ❌ 不扩展 quickstartFunctions |
| 微信登录 | NestJS code2Session | — |
| 快速演示 | — | 仅 MVP 第一周可选，不双写 |

**原则：避免「云开发 + 自建 API」双写同一业务表**，否则同步与一致性成本高于收益。

## 七、分期落地（技术视角）

### Phase 1 — MVP（4–6 周）

- [ ] Monorepo 脚手架 + Docker 本地 PostgreSQL
- [ ] NestJS：微信登录、用户/角色、课程 CRUD、问卷提交与同步
- [ ] Taro：4 Tab 骨架、关怀模式、课程列表/详情、问卷离线填写
- [ ] Admin：登录、课程 CMS、问卷数据列表与导出
- [ ] COS：视频/封面上传链路打通
- [ ] 隐私政策与用户协议页面（小程序 + 后台可配置链接）

### Phase 2 — 数据闭环

- [ ] 学情前后测对比报表 API + 后台图表
- [ ] Redis + 订阅消息（微信模板消息经后端触发）
- [ ] 调研库批量脱敏导出、操作审计日志
- [ ] OpenAPI 文档对外（队员/校方查阅）

### Phase 3 — 智能化与生态

- [ ] 教具扫码 / 蓝牙（视硬件，独立模块接入 API）
- [ ] 文博 FAQ / 大模型（类目与内容审核后接入）
- [ ] Taro 编译 H5（成果展示专题页，可选）

## 九、风险与对策

| 风险 | 对策 |
|------|------|
| Taro 与微信新特性滞后 | 关键能力（录音、订阅消息）用 Taro 插件或 `nativeComponents` |
| 校内服务器审批慢 | 开发期用腾讯云轻量 + Docker；数据迁移脚本就绪 |
| 前后端两人不足 | `packages/shared` 统一类型；Swagger 自动生成减少联调成本 |
| 关怀模式体验 | 独立主题与页面，不与普通模式共用复杂布局 |
